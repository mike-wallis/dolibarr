How To — Correct a Wrong Price on a Purchase Order

Dolibarr version 23.0.3. Applies to Purchase Orders (Supplier Orders) at any stage before the supplier invoice has been entered. Worked example below uses real steps carried out on PO2607-0001 (Hanleys) on 08/07/2026.

Golden rule: Dolibarr only lets you edit a purchase order line's price while the order status is Draft. If the order has moved past Draft (Approved, Ordered, Received…), you must step it back to Draft first, fix the price, then move it forward through the workflow again.

The full PO workflow, and where you can/can't edit prices

Status Can you edit the price directly?
Draft Yes — prices fully editable
Approved No — locked
Ordered No — locked
Received No — locked, and stock cost is now posted
Invoiced No — but the PO itself no longer matters at this point

Once a PO is Approved or later, the "Add/edit line" form disappears from the page entirely — there is no direct way to change a unit price. You have to use one of the buttons on the order card to send it back a step.

How to send an order back to Draft

Current status Button to click What happens
Approved REJECT (also shown as "Disapprove" in some Dolibarr configs) Goes straight back to Draft in one click.
Ordered / Partially received / Received REJECT / ReOpen Only moves back one status at a time (e.g. Received → Ordered → Approved). Click it repeatedly, and once it's back at Approved, click it once more to reach Draft.

The exact button label can vary (we've seen both "Disapprove" and "REJECT" for the same action on this Dolibarr install — it depends on the site's language pack). Whatever it's called, Dolibarr always shows a confirmation popup before anything actually changes — read the popup wording before confirming. If it talks about the order being sent back for correction, that's the right one. If it talks about the supplier refusing/declining to fulfil the order, that's a different action — cancel out and don't use it.

Once you're back at Draft

  1. Click the pencil (✎) icon on the right of the line with the wrong price.
  2. Correct the U.P. (net) unit price.
  3. Save the line. Repeat for any other wrong lines.
  4. Check the totals at the top right of the page have recalculated correctly.
  5. Click Validate and Approve (or Validate, then Approve, depending on the order's value/workflow) to send it forward again.
  6. Continue the normal workflow from there: Make Order → record the Item receipt → Create Invoice.

⚠ If the goods were already received in Dolibarr with the wrong price

Recording a goods receipt posts a permanent stock movement using whatever price was on the order line at that moment, and immediately updates the product's weighted-average cost (AVCO). That stock movement is not linked back to the order line — correcting the price on the PO afterwards will fix the PO document, but it will not retroactively fix the stock value already posted.

If this happens: fixing the PO is still worth doing for accurate records, but the stock valuation itself needs a separate manual correction (Products → the item → Stock tab → Correct Stock). This touches inventory valuation, so check the size of the discrepancy and, if it's material, flag it for the accountant/BAS agent rather than guessing at the right adjustment.

If the supplier invoice has already been entered in Dolibarr

You don't need to touch the PO at all. A supplier invoice's line prices don't have to match the PO — enter the correct prices straight from the vendor's actual invoice. The invoice (not the PO) is what matters for accounts payable and GST/BAS. If the invoice was already validated with the wrong price, it needs a credit note or a Modify/correction on the invoice itself, not a fix on the PO.

Worked example — PO2607-0001 (Hanleys)

Two lines had the wrong unit price after the order was already Approved, but before "Make Order" or any goods receipt had been recorded in Dolibarr — the simplest case.

Product Wrong price (net) Correct price (net)
AUPR240LT — Wheelie Bin Liner 240 Lt Black $38.00 $37.01
CC0051879PK — BLEACH 4% 5 Lt $12.83 $9.41
  1. On the PO card (status: Approved), clicked REJECT — confirmed via the popup — order returned to Draft (needs to be validated).
  2. Edited the two wrong lines using the pencil icon, corrected the unit prices.
  3. Clicked VALIDATE AND APPROVE — order returned to Approved, totals recalculated to $585.78 excl. tax / $644.36 inc. tax.
  4. Regenerated the PO PDF (Linked files tab → Generate) — confirmed it showed the corrected prices.
  5. Clicked Make Order, then went to Item receipts and clicked Create Reception for the full quantities (16 / 4 / 6), confirmed the reception, then clicked Validate on the reception.
  6. Checked the product's Stock tab (e.g. AUPR240LT) — Weighted average price now $37.01, correctly reflecting the fixed price, since the reception hadn't happened yet when the fix was made.
  7. Clicked Create Invoice from the PO — the supplier invoice (SI2607-0001) picked up the corrected prices automatically, totalling $644.36 inc. tax.
  8. Checked the vendor reference, invoice date, and line-by-line amounts against Hanleys' actual paper invoice before validating the invoice.

Quick reference — what to check before you validate anything

  • Totals at top of the PO/invoice match what you expect after the edit.
  • Regenerate the PDF if you're about to send/re-send it — don't rely on a previously generated copy.
  • If stock has already been received with a wrong price, the PO fix alone is not enough — see the warning above.
  • Supplier invoice line prices are independent of the PO — always compare the final invoice against the vendor's actual paper/PDF invoice before validating, regardless of whether the PO was ever wrong.
